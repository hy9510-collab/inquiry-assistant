# 의정활동 비서 (경기도의회 문화체육관광위원회)

## 프로젝트 개요
경기도의회 문체위 정책지원관이 담당 의원 2명의 의정활동을 지원·기록·공유하는 시스템.
전체 계획: `01_기획/의정활동비서_기획서.docx` 참조.

## 폴더 구조
- `01_기획/` — 기획서, 설계 문서
- `02_페이퍼/` — 온보딩 페이퍼 3종 (문체위 브리핑, 초선용 안내, 의원 프로필)
- `03_대장/` — 데이터 원본 (공약·민원·의회활동·보도·일정, xlsx/JSON)
- `04_리포트/` — 의원 공유용 리포트 (회차별 아카이브, hwpx)

## 작업 원칙
1. 데이터 원본은 03_대장에 한 곳만 유지 ("원본 하나, 출력 여러 개")
2. 모든 활동 기록에 날짜·출처(원문 링크) 병기
3. 민원인 정보는 비식별 처리 (이니셜·유형만)
4. 공약은 선관위 선거공보 원문 기준
5. 의원 공유 문서는 hwpx 최종본, 파일명: `리포트_YYYYMM_의원명.hwpx`

## 데이터 소스
- 경기도의회: https://www.ggc.go.kr (의원소개, 의안)
- 회의록시스템: https://kms.ggc.go.kr (의원별 발언 검색)
- 선관위 선거통계시스템 (공약 원문)

## Claude Code 개발 과제 (4단계)
1. **hwpx 생성기**: `04_리포트/템플릿.hwpx`(OWPML zip)의 본문 XML을 치환해 회차별 리포트 생성
2. **회의록 수집 스크립트**: kms.ggc.go.kr 의원별 발언 목록·링크 수집 → 03_대장에 저장
3. **대장 스키마**: 공약(상태: 구상/추진중/일부이행/완료/보류), 민원(접수/검토/이송/처리/회신)
4. **대시보드 고도화**: 03_대장 데이터를 읽는 열람용 HTML
